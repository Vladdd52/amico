# Amico.uz — Интернет-магазин сумок и аксессуаров

> Современный интернет-магазин для малого бизнеса, разработанный на Django с поддержкой оформления заказов через Telegram и Instagram.

🌐 **Сайт:** [amico.uz](https://amico.uz)

---

## О проекте

**Amico.uz** — полноценный интернет-магазин с каталогом товаров, корзиной и оформлением заказов. Разработан специально для малого бизнеса, которому нужна красивая витрина с удобным управлением через стандартную Django-админку, без лишней сложности.

Покупатель выбирает товар, добавляет в корзину и оформляет заказ через Telegram или Instagram — менеджер получает готовое сообщение со всеми деталями. Простая и рабочая схема для старта.

Архитектура намеренно спроектирована с заделом на рост: в будущем можно подключить платёжные системы (Click, Payme) без переработки основного кода.

## Быстрый старт

### 1. Клонировать репозиторий

```bash
git clone https://github.com/Vladdd52/amico.git
cd amico
```

### 2. Создать виртуальное окружение и установить зависимости

```bash
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 3. Создать файл `.env`

```env
SECRET_KEY=ваш-секретный-ключ
DEBUG=True
ALLOWED_HOSTS=localhost,127.0.0.1

POSTGRES_DB=amico
POSTGRES_USER=postgres
POSTGRES_PASSWORD=ваш-пароль
POSTGRES_HOST=localhost
POSTGRES_PORT=5432

ADMIN_PATH=admin/
SECURE_SSL=False
```

### 4. Применить миграции

```bash
python manage.py migrate
```

### 5. Создать суперпользователя

```bash
python manage.py createsuperuser
```

### 6. Запустить сервер

```bash
python manage.py runserver
```

Сайт доступен по адресу: `http://127.0.0.1:8000`  
Админка: `http://127.0.0.1:8000/admin/`

## Безопасность

- **CSRF-защита** — на всех формах
- **django-axes** — блокировка после 3 неудачных попыток входа на 30 минут
- **django-ratelimit** — ограничение 10 запросов/минуту на все операции с корзиной и заказами
- **XSS** — экранирование через шаблонизатор Django + заголовок `X-XSS-Protection`
- **Clickjacking** — заголовок `X-Frame-Options: DENY`
- **SQL-инъекции** — исключены через ORM Django
- **Секреты в .env** — не попадают в репозиторий
- **Нестандартный URL админки** — задаётся через `ADMIN_PATH`
- **ATOMIC_REQUESTS** — каждый запрос в транзакции БД
- **HTTPS / HSTS** — включаются через `SECURE_SSL=True`

## HTMX и UX

Сайт использует HTMX для динамических обновлений без перезагрузки страницы:

- Навигация по каталогу обновляет только блок с товарами
- Фильтры применяются мгновенно
- Корзина обновляется без потери позиции скролла
- Иконка корзины в шапке обновляется через OOB-своп
- Переключение цвета товара обновляет галерею через OOB-своп

## Деплой на ahost.uz (cPanel)

1. Загрузить файлы проекта через File Manager или FTP
2. Создать виртуальное окружение и установить зависимости
3. Создать базу данных PostgreSQL в cPanel
4. Настроить `.env` с боевыми данными
5. Запустить `python manage.py migrate` и `python manage.py collectstatic`
6. Настроить Python App в cPanel, указав `passenger_wsgi.py` как точку входа
7. Привязать домен
